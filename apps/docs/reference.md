# Technical Reference

Complete specification for the Primust verification infrastructure ‚Äî invariants, schemas, APIs, and gap taxonomy.

---

## System Invariants

The following invariants hold for every Primust deployment. Violation of any invariant is treated as a system failure.

1. **No data transit.** Raw content (inputs, outputs, PII, model weights) NEVER leaves the customer environment. Only commitment hashes (Poseidon2 or SHA-256) transit to `api.primust.com`.
2. **Append-only chain.** Once a check execution record is committed, it cannot be modified or removed. The chain is hash-linked ‚Äî altering any record invalidates all subsequent chain hashes.
3. **Weakest-link proof level.** `proof_level_floor` is the lowest proof level across all records in the run. Cannot be faked upward. DERIVED ‚Äî never set manually.
4. **Gaps recorded honestly.** Every known gap must be disclosed in the VPEC using the canonical gap taxonomy. Undisclosed gaps constitute a system integrity failure. `detectable_from_surface: false` means "unknown" not "ungoverned."
5. **Offline verifiability.** Any VPEC can be verified without network access using `primust verify --trust-root`, given the signing public key.
6. **Content-blind by default.** `visibility: "opaque"` is the default on all records. All connector records enforce opaque ‚Äî not configurable by caller.
7. **Fail open, fail honest.** Primust failures do not block customer pipelines. All failures become gaps.
8. **Schema forward compatibility.** Newer versions of `primust-verify` can verify VPECs generated by older schema versions. Current schema version: `5.0.0`.
9. **Primust never holds reviewer credentials.** Reviewer Ed25519 private key never leaves customer environment.
10. **human_review maps to Witnessed, not Attestation.** Protocol distinction, not a label.
11. **Manifest hash captured per record.** Every CheckExecutionRecord stores `manifest_hash` at time of execution.
12. **upstream_vpec_verify proof ceiling is Mathematical.** Ed25519 verification is deterministic. Never downgrade this ceiling.

## VPEC Schema (v5.0.0)

The Verifiable Process Execution Credential. Issued when a run is closed.

```json
{
  "vpec_id": "vpec_a1b2c3...",
  "schema_version": "5.0.0",
  "org_id": "org_x9y8z7",
  "run_id": "run_8f3a2b...",
  "workflow_id": "loan-underwriting-v3",
  "policy_snapshot_hash": "sha256:112233...",
  "environment": "production",
  "partial": false,
  "state": "signed",
  "proof_level_floor": "execution",
  "provable_surface": 0.73,
  "provable_surface_breakdown": {
    "mathematical": 0.62,
    "verifiable_inference": 0.00,
    "execution": 0.11,
    "witnessed": 0.00,
    "attestation": 0.00
  },
  "provable_surface_pending": 0.00,
  "provable_surface_ungoverned": 0.27,
  "provable_surface_basis": "executed_records",
  "provable_surface_suppressed": false,
  "gaps": [],
  "issuer": {
    "issuer_id": "primust.com",
    "issuer_type": "platform"
  },
  "signature": {
    "algorithm": "Ed25519",
    "public_key_id": "key_m3n4o5",
    "value": "base64..."
  },
  "timestamp_anchor": {
    "method": "rfc3161",
    "value": "base64..."
  },
  "issued_at": "2026-03-16T09:14:00Z"
}
```

**Key field reference:**

| Field | Type | Notes |
|---|---|---|
| `proof_level_floor` | enum | Weakest-link scalar. DERIVED. Never set manually. |
| `provable_surface` | float 0.0‚Äì1.0 | Hero metric. Not a percentage. |
| `provable_surface_breakdown` | object | Five sub-fields, all floats, sum = `provable_surface` ¬±0.0001 |
| `environment` | `sandbox` \| `production` | Present on every VPEC |
| `gaps` | array | Canonical name ‚Äî `governance_gaps` is banned |

**Banned field names (schema v5.0.0 removes these):**

| Banned | Use Instead |
|---|---|
| `proof_level` (VPEC-level) | `proof_level_floor` |
| `proof_distribution` / `proof_level_breakdown` | `provable_surface_breakdown` |
| `coverage_verified_pct` | `provable_surface` |
| `governance_gaps` | `gaps` |
| `test_mode: true` | `environment: "sandbox"` |

## Check Execution Record Schema

Each call to `p.record()` produces one check execution record.

```json
{
  "record_id": "rec_f1g2h3...",
  "run_id": "run_8f3a2b...",
  "manifest_id": "sha256:aabb...",
  "manifest_hash": "sha256:aabb...",
  "commitment_hash": "poseidon2:a1b2c3...",
  "output_commitment": "poseidon2:e5f6g7...",
  "commitment_algorithm": "poseidon2",
  "commitment_type": "input_commitment",
  "check_result": "pass",
  "proof_level_achieved": "execution",
  "visibility": "opaque",
  "check_open_tst": "2026-03-16T09:14:18.003Z",
  "check_close_tst": "2026-03-16T09:14:18.145Z",
  "chain_hash": "sha256:q7r8s9...",
  "idempotency_key": "idem_a1b2c3...",
  "actor_id": null,
  "explanation_commitment": null,
  "bias_audit": null
}
```

| Field | Valid Values |
|---|---|
| `check_result` | `pass`, `fail`, `error`, `skipped`, `degraded`, `override`, `not_applicable`, `timed_out` |
| `proof_level_achieved` | `mathematical`, `verifiable_inference`, `execution`, `witnessed`, `attestation` |
| `commitment_algorithm` | `poseidon2`, `sha256` |
| `commitment_type` | `input_commitment`, `metadata_commitment`, `foreign_event_commitment` |
| `visibility` | `opaque` (default), `selective`, `transparent` |

## API Endpoints

Base URL: `https://api.primust.com`. SDK endpoints use `X-API-Key` header (`pk_sb_*` sandbox, `pk_live_*` production).

### Run Lifecycle (SDK)

| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/v1/runs` | Start a new run. Returns `run_id`, `policy_snapshot_hash`. |
| GET | `/api/v1/runs/:id` | Get run status and summary. |
| POST | `/api/v1/runs/:id/records` | Commit a check execution record. Returns `record_id`, `chain_hash`. |
| POST | `/api/v1/runs/:id/close` | Close and seal the run ‚Äî triggers VPEC issuance. Returns VPEC. |

### VPECs

| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/v1/vpecs/:id` | Retrieve a VPEC by ID. |

### Evidence Packs and Audit Reports

| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/v1/packs` | Assemble an Evidence Pack from VPEC IDs. Returns `pack_id`, `merkle_root`. |
| POST | `/api/v1/evidence-packs/:pack_id/report` | Generate signed PDF audit report. |
| GET | `/api/v1/reports/:id` | Download signed audit report. 90-day TTL. |

### Policy Bundles and Manifests

| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/v1/policy/bundles` | List all bundles (7 built-in + org custom). |
| POST | `/api/v1/manifests` | Register a manifest (content-addressed, idempotent). |
| GET | `/api/v1/manifests/:id` | Get a manifest (public, no auth). |

### BYOK Signing

| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/v1/org/signing-keys` | Register Ed25519 BYOK key. |
| POST | `/api/v1/org/signing-keys/:id/verify` | Challenge-response verification. |
| DELETE | `/api/v1/org/signing-keys/:id` | Revoke key (historical VPECs remain verifiable). |

### Public Key Distribution

| Method | Endpoint | Description |
|---|---|---|
| GET | `/.well-known/primust-pubkeys/:kid.pem` | Public key by key ID. Used by `primust verify` for online key resolution. |

### Gaps and Waivers

| Method | Endpoint | Description |
|---|---|---|
| GET | `/api/v1/gaps` | List gaps filtered by `run_id`, `state`, `severity`. |
| POST | `/api/v1/gaps/:id/waive` | Create a waiver. `risk_treatment` required ‚Äî no default. |

### Webhooks (SIEM Integration)

| Method | Endpoint | Description |
|---|---|---|
| POST | `/api/v1/webhook` | Create or update SIEM webhook. |
| POST | `/api/v1/webhook/test` | Send test event to configured endpoint. |
| GET | `/api/v1/webhook/failures` | List last 50 dead-letter entries. |
| POST | `/api/v1/webhook/retry/:delivery_id` | Replay a failed delivery. |

## Gap Taxonomy ‚Äî 45 Types

### Core (22)

| `gap_type` | Severity | Description |
|---|---|---|
| `check_not_executed` | Critical | A declared check was not executed during the run |
| `enforcement_override` | High | Check enforcement was overridden |
| `engine_error` | High | Governance engine encountered an error |
| `check_degraded` | Medium | Check ran in reduced/degraded mode |
| `external_boundary_traversal` | High | Data crossed external boundary without governance |
| `lineage_token_missing` | High | Delegation handoff missing lineage token |
| `admission_gate_override` | High | Admission gate bypassed or overridden |
| `check_timing_suspect` | Medium | Review duration below minimum ‚Äî possible rubber-stamping |
| `reviewer_credential_invalid` | Critical | Reviewer credential failed validation |
| `witnessed_display_missing` | High | Witnessed record lacks `display_hash` |
| `witnessed_rationale_missing` | Medium | Witnessed record lacks `rationale_hash` |
| `witnessed_timestamp_invalid` | High | Witnessed record has invalid timestamp |
| `deterministic_consistency_violation` | Critical | Same input produced different results across runs |
| `skip_rationale_missing` | Medium | Check skipped without `skip_rationale_hash` |
| `policy_config_drift` | Medium | Policy config changed between manifest snapshot and execution |
| `proof_level_floor_breach` | Critical | Record's proof level below policy bundle minimum |
| `zkml_proof_pending_timeout` | Medium | ZKML proof generation timed out |
| `zkml_proof_failed` | High | ZKML proof generation failed |
| `system_error` | High | Unrecoverable error during governance processing (Primust-side) |
| `sla_breach` | Medium | Governance SLA target not met |
| `explanation_missing` | Medium | Decision lacks explanation commitment ‚Äî fires only when `compliance_requirements.require_explanation_commitment` set |
| `bias_audit_missing` | High | Decision on protected categories lacks bias audit ‚Äî fires only when `compliance_requirements.require_bias_audit` set |

### System Availability (1)

| `gap_type` | Severity | Description |
|---|---|---|
| `system_unavailable` | High | Primust API unreachable ‚Äî SDK queued locally, queue lost or TTL expired. Distinct from `system_error` (processing failure). |

### Unstructured Check (1)

| `gap_type` | Severity | Description |
|---|---|---|
| `archetype_unmapped` | Medium | `custom_check` VPEC exists for check_id but compliance officer has not mapped it. One gap per unique unmapped check_id. Auto-resolves when officer completes mapping. |

### Cross-Org Verification (7)

| `gap_type` | Severity | Trigger |
|---|---|---|
| `upstream_vpec_missing` | High | No upstream VPEC delivered with upstream output |
| `upstream_vpec_invalid_signature` | Critical | Ed25519 verification failed |
| `upstream_vpec_sandbox` | High | Upstream VPEC is sandbox in a production context |
| `upstream_vpec_key_revoked` | High | Upstream signing key revoked |
| `upstream_vpec_insufficient_proof_level` | High | Upstream `proof_level_floor` below declared minimum |
| `upstream_vpec_missing_claim` | Medium | Required claim absent from upstream VPEC |
| `upstream_vpec_issuer_mismatch` | Critical | Upstream issuer does not match `expected_issuer_org_id` |

### Connector-Specific (14)

Pattern: `{platform}_api_error` (High) = vendor API unreachable or 5xx. `{platform}_auth_failure` (Critical) = vendor API 401/403.

`complyadvantage_api_error`, `complyadvantage_auth_failure`, `actimize_api_error`, `actimize_auth_failure`, `blaze_api_error`, `blaze_auth_failure`, `odm_api_error`, `odm_auth_failure`, `falcon_api_error`, `falcon_auth_failure`, `pega_api_error`, `pega_auth_failure`, `wolters_kluwer_api_error`, `wolters_kluwer_auth_failure`

> ‚öÝ check_candidate findings are NOT gap types. They are scan advisories. They never appear in VPEC `gaps` and never affect `provable_surface`.

## Verification

### Four Verification Paths (in priority order)

**Path 0 ‚Äî primust-verify embedded (highest frequency ‚Äî relying party)**
```python
from primust_verify import verify

result = verify(vpec_json)
result = verify(vpec_json, trust_root_pem=KEY_PEM)   # zero network

if not result.valid:
    raise GovernanceVerificationError(result.failure_reason)
```

**Path 1 ‚Äî primust-verify CLI (Apache-2.0)**
```bash
pip install primust-verify
primust verify vpec.json
primust verify vpec.json --trust-root key.pem
primust pack verify pack.json
```

**Path 2 ‚Äî primust verify CLI (SDK bundled)**
Ships with main SDK. Same library as Path 1.

**Path 3 ‚Äî verify.primust.com**
Human front door for auditors. Convenience only ‚Äî not the trust anchor.
Page displays: "You don't need this website. pip install primust-verify ‚Äî works offline, forever."

### Exit Codes

| Code | Meaning |
|---|---|
| 0 | PASS ‚Äî all verification checks passed |
| 1 | FAIL ‚Äî one or more checks failed |
| 2 | SANDBOX ‚Äî valid proof, not audit-acceptable |
| 3 | KEY REVOKED ‚Äî valid proof, signing key revoked |
